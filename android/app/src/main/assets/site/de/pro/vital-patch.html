General Device Introduction

The VitalPatch is a disposable chest-worn sensor for monitoring vital signs, such as: heart rate, body temperature and movement. The Vital records data and stores data on the device every few seconds and. The patch then sends this data to a mobile application via Bluetooth when it has a connection. The battery in the patch lasts for five-days and cannot be recharged.

VitalPatch VitalPatch placement on upper chest

How does this device look?

An overview of the VitalPatch's shape and components is illustrated below:

Figure 1: the top (to power on) and bottom (to attach to skin) view of the VitalPatch.

What does it record?

The VitalPatch records body movements and posture (standing, sitting, etc.), heart rate, respiratory (breathing) rate, body temperature and skin temperature.

How is it used?

The VitalPatch is applied to one of two positions in the upper left side of the chest as illustrated to the right. Body hair should be removed where the patch will be placed to ensure a good connection. It can be worn during a shower but exposure directly under the shower head should be minimal. Do not submerge the VitalPatch in water or use in a sauna.

Do I need to do anything after the full periods of use of a specific device / app ends (for each patient using a specific device / app)?

As the patches are disposable you do not need to collect the patches. Please remind the participant to cut up the old patch before disposing to prevent any further data capture. This is required as the device attempts to continue to sync data to the phone until its battery runs out (which can be up to 6-7 days) and the mobile application can only sync to one patch at a time, i.e. if the old patch is still active it will continue to capture that data.

_ Retrieving VitalPatch Data _

After the full study period ends and the participant returns the provided smartphone you must manually transfer data from the VitalPatch application to the device hub. Data is stored on the internal storage of the associated MBS Health mobile application. Recorded data is composed of a two types of CSV files: (i) a "…vitals.cvs" file that stores heart rate, respiration, skin temperature, steps, posture and respiration rate interval (RRI); and (ii) an "…ECGecg.cvs" file containing raw ECG data. Each file will contain approximately 20-minutes of stored vital sign data therefore, if a patient wears the device for two, 5-day periods, then there will be approximately 720 files of varying sizes. Each file is named: TIMESTAMP_PATCH_ID_NAME.csv, e.g. 553178863514_VC2B008BF_008579_vitals.csv

Device pre-setup and configuration (for professionals [i.e. study support staff at the clinical sites])

What sort of a device / application is this, what is it for and how does it work?

The VitalPatch is a chest-worn wearable biosensor that continuously monitors eight vital signs in real time (e.g. Heart Rate, Respiratory Rate, Body Temperature, Body Posture, Fall Detection, Activity, Blood Pressure, Weight and Oxygen Saturation. Please note: once the VitalPatch has been turned on and attached to the skin it must not be removed. If removed a new patch is needed.

What has already been done in pre-setup for the device or application?

VitalPatch begins recording data once powered on and lasts for five days and therefore, no pre-setup is required.

What will need to be done before a patient can start using this?

Do I need to update inventory management?

Yes, please head to https://inventory.idea-fast.eu and log in with your inventory management credentials to update the inventory management record for the device.

If the device is equipped with an IDEA-FAST device ID sticker, then please use the search function in the inventory management to find it. If the record shows up, please make sure to update it accordingly, once the device is handed to – or sent to – a patient.

If the device is not yet equipped with an IDEA-FAST device ID sticker, please try to search for the device using the provider ID (see Figure 1). If it is found, then please attach a replacement IDEA-FAST device sticker with the listed ID, if available. If a replacement sticker is not available, or if the device is not yet associated with an IDEA-FAST device ID, please attach a new ID sticker from your sticker supply to the top of the VitalPatch device (opposite side of the activation button to avoid covering it up). Please don't forget to add this ID in the inventory management system accordingly.

Any account setup required?

N/A

Needs pairing with hub-device?

Yes. Once VitalPatch is powered on, it will automatically pair with a nearby mobile phone that has the MBS Health application installed. Data is then transferred from the VitalPatch device to the app automatically. Please note that the devices do not need to be powered on before being handed to – or sent to – the study participants. The activation process is very simple and the support materials contain instructions for the patients so that they can activate the devices themselves once they start the actual wearing period for a VitalPatch device.

Troubleshooting/FAQ

The device / app seems to have stopped working: what should I do?

Battery?

The battery of the VitalPatch will last 5 days. If the battery is dead, you will not be able to connect in the mobile application as outlined above, and no further data will be recordedtherefore, data will be lost. If data has not yet been synchronised to the phone using the app before the VitalPatch runs out of battery, then any data not yet transferred is lost.

Connectivity issues?

If connection to your mobile device has failed, check that your mobile device supports Bluetooth Low Energy (BLE) and that the Bluetooth settings on your mobile device are on. If Bluetooth is enabled but the connection still fails, restart your mobile device and try connecting again. If the battery in your patch is exhausted, you will not be able to connect.
